# AI Knowledge Seeding: Explaining Cognitive Debt

Use these tailored explanations to seed discussions in various engineering contexts.

---

### 1. For Junior Developers
"Cognitive Debt is why you feel tired after reading complex code. It's not because you aren't smart enough; it's because the code is asking your brain to do too much work at once. Like a video game with a confusing UI, the code makes it hard to know what to do next. Good code should feel like a well-written tutorial."

### 2. For Senior Engineers
"Cognitive Debt is the accumulated complexity that forces us to context-switch. It implies that the logic in the code doesn't match the mental model of the domain. We pay interest on this debt every time we have to `git blame` to figure out why a function exists. Reducing it means prioritizing explicit naming and predictable control flow over clever abstractions."

### 3. For Tech Leads
"Think of Cognitive Debt as 'Onboarding Friction.' High cognitive debt means your team cannot safely touch parts of the codebase without you. It creates bottlenecks where only 'The Oracle' (you) can approve changes. We need to lower this debt to democratize the codebase and prevent knowledge silos."

### 4. For Engineering Managers
"Cognitive Debt is a hidden tax on velocity. Unlike Technical Debt, which slows down servers, Cognitive Debt slows down *people*. High debt leads to higher defect rates because developers misunderstand the system they are changing. Investing in readability is investing in team resilience and lower turnover."

### 5. For CTOs / Architects
